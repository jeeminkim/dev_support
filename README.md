# K-Dev Assistant

K-Dev Assistant는 사내 개발자 및 기획자의 업무를 보조하기 위해 만들어진 AI 어시스턴트 MVP입니다. 자연어 요구사항을 바탕으로 순서도 시각화, SQL 작성, TypeScript 코드 뼈대 생성을 빠르게 지원합니다.

## ✨ 주요 기능
- **Flow 시각화**: 자연어 비즈니스 로직을 파악하고 Mermaid 문법 기반의 다이어그램과 설명을 제공합니다.
- **SQL 생성**: 사내 DB 환경(PostgreSQL, MySQL, Oracle)에 맞춰 초기 쿼리와 DDL/DML 예시를 생성합니다.
- **TypeScript 뼈대 생성**: 명확한 타입이 지정된 구조 및 유틸리티 뼈대 코드를 제공합니다.
- **후속 수정(Follow-up)**: 생성된 결과물을 바탕으로 지속적인 대화를 통해 점진적으로 코드를 고도화할 수 있습니다.
- **Local Storage 우선**: 안전한 MVP 테스트를 위해 히스토리 및 워크스페이스 상태는 브라우저 내부(Local Storage)에 저장되며 서버에 보관되지 않습니다.

## 🚀 로컬 구동 방법
이 프로젝트는 [Next.js](https://nextjs.org) 14/15 환경과 React 19을 기반으로 합니다.

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```
이후 브라우저에서 [http://localhost:3000](http://localhost:3000) 로 접속합니다.

## ⚙️ 설정 가이드 및 한계점
* 현재 MVP는 인증/백엔드 DB 없이 완전한 Client-Side Storage로 운영됩니다.
* **API Key 설정**: 화면 우측 상단의 `설정(톱니바퀴)` 메뉴에서 Gemini API Key를 입력해야 정상 동작합니다.
* **보안 한계**: 로컬 스토리지에 API 키가 암호화되지 않은 상태로 임시 저장됩니다. 따라서 공용 PC에서는 사용 후 데이터 초기화 버튼을 이용하거나, 가급적 개인 환경에서만 테스트용으로 사용 바랍니다.

## 🔜 향후 배포 및 2차 개선 방향
1. **서버 기반 키 관리**: 브라우저 직접 저장이 아닌 서버 환경 변수를 통한 프록시 보안 처리
2. **사내 DB 연동 플러그인**: 단순 코드 생성을 넘어 스키마 정보를 연동한 맞춤형 생성 고도화
3. **유저 인증 및 히스토리 관리**: 부서별 활성화를 위한 RDBMS 연동 및 OAuth 통합
